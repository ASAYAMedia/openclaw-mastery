Remembering is crucial for an AI assistant. In this guide, you'll set up a fully private, local memory system using Nomic Embeddings and OpenClaw's memory tools.

Why Local Embeddings?

  • üîí Privacy: Your data never leaves your machine.
  • üöÄ Performance: Fast retrieval without network latency.
  • üí∞ Cost-Effective: No API fees for embeddings.

Step 1: Download Nomic Embeddings Model

Model Details

We'll use nomic-embed-text-v1.f16.gguf. It's a powerful, quantized model suitable for local use.

Download Link

From Hugging Face: Download GGUF Model

Installation

Save the downloaded file to a known location. For example:

C:\Temp\nomic-embed-text-v1.f16.gguf
üí° Model Size: This model is around 274MB. Ensure you have enough disk space.

Step 2: Configure OpenClaw Memory

Edit your OpenClaw config file (`~/.openclaw/openclaw.json`).

Update `memorySearch` Section

Add or modify the `memorySearch` section to point to your local model.:


{
  "agents": {
    "defaults": {
      "memorySearch": {
        "enabled": true,
        "provider": "local",
        "local": {
          "modelPath": "C:\\Temp\\nomic-embed-text-v1.f16.gguf"
        },
        "query": {
          "minScore": 0.7,
          "maxResults": 10
        }
      }
    }
  }
}
‚ö†Ô∏è Paths: Ensure `modelPath` exactly matches where you saved the Nomic Embeddings model file.

Step 3: Test Memory Retrieval

OpenClaw automatically uses the configured memory. To test, you need to populate it first. Try sending a few messages, then search for them:

# After sending some messages to your bot...

# Search for a specific topic
/memory_search "how to set up discord bots"

# Ask a question that requires recalling past info
/ask "What was my last instruction regarding income generation?"

You should see relevant snippets from your conversation history. If `memorySearch` is `enabled: true`, OpenClaw will use this local memory automatically before answering.
